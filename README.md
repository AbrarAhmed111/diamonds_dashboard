# Diamond Pigs Signals Dashboard — POC

A pixel-perfect, **frontend-only** proof of concept of the Diamond Pigs Signals
Dashboard. Built to validate the look, feel, and information architecture of
the product against a static `indicators.json` payload (no backend required).

> See [`POC_INFO.md`](./POC_INFO.md) for delivery / hosting notes and
> [`instructions_for_cursor/instructions.md`](./instructions_for_cursor/instructions.md)
> for the original brief.

## Stack

- Next.js 15 (App Router, static export)
- TypeScript + Tailwind CSS (design tokens mirroring the Figma scale)
- Recharts for the sparkline and detail charts
- Lucide React icons
- Inter Tight via `next/font/google`

## Routes

| Route       | Purpose                                                                |
| ----------- | ---------------------------------------------------------------------- |
| `/`         | Overview dashboard — market sentiment summary + signal grid + drilldown |
| `/signals`  | Full signal explorer with search, filters, sort, and detail panel       |
| `/settings` | Static POC info: data source, refresh, deployment notes                 |

## Sentiment theming

The dashboard reads the overall sentiment from the loaded signals (majority
rule, fall-back to `neutral`) and exposes it to CSS via a `data-sentiment`
attribute on the app shell. CSS variables (`--sentiment-rgb`,
`--sentiment-soft-rgb`, `--sentiment-ink-rgb`, `--sentiment-glow`) flip with
the theme, so badges, summary surfaces, charts, and accents recolor without
re-rendering layout.

## Accessibility highlights

- WCAG 2.2 AA targets across color contrast, focus rings, and keyboard
  navigation.
- All interactive surfaces expose default / hover / focus-visible / active /
  disabled states.
- Status and sentiment badges combine color, icon, and text label so they
  never rely on color alone.
- Sparkline and detail charts ship with `role="img"` and descriptive
  `aria-label`s; empty datasets render readable fallback text.

## Updating data

1. Run the backend export pipeline (`python indicators_export.py`).
2. Replace `public/data/indicators.json` with the freshly generated file.
3. Redeploy (or upload the file to your static host).
4. The dashboard refetches automatically on next load and respects a manual
   refresh button on every page.
